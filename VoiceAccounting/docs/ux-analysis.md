# UX 分析（语音记账 iOS）

日期：2026-02-07  
执行者：Codex

## 核心用户需求
- **快速记录**：语音输入与手动输入并存，减少记录成本。
- **低认知负担**：界面层级少，入口集中，常用操作靠近拇指区。
- **即时反馈**：录入后及时在“今日记录”中可见。
- **可回溯**：历史按天分组，便于找回与核对。

## 关键使用场景与交互逻辑
1. **即时记账**
   - 入口：主页双按钮（语音/手动）。
   - 语音：长按录音 → 语音识别 → 自动建议金额/分类 → 用户确认。
   - 手动：弹出底部弹窗，输入金额、标题、时间、分类。
2. **趋势查看**
   - 统计页切换月/季度/年。
   - 趋势图按时间顺序展示；分布图按分类降序。
3. **历史回看**
   - 按天分组，快速定位特定日期消费。
4. **设置维护**
   - 货币单位设置、分类管理（增删改查）。
   - 数据管理（导出 CSV、清空数据）。

## 播客应用行为研究对交互的启发
> 需求中明确要求参考“播客应用用户行为（发现、播放、订阅、社交分享）”，以下做等价映射以指导语音记账体验。

- **发现 → 快速入口**：播客用户常从常用应用内发现新内容（来源：The Podcast Host 2024 调研，https://www.thepodcasthost.com/listening/100-podcast-statistics/），因此语音记账首页将“语音输入/手动输入”作为一级入口。
- **播放 → 录音体验**：播放强调即时性，映射为“长按录音”与实时波形反馈。
- **订阅 → 分类偏好**：播客订阅习惯对应“常用分类优先显示”，减少重复选择成本。
- **分享 → 导出/复盘**：社交分享映射为“导出 CSV”，便于和他人共享或复盘。

## 成功指标（可用于后续验证）
- 记账完成时间 < 10 秒（语音场景）
- 记账入口点击路径 ≤ 1 步
- 历史查找时间 ≤ 15 秒
